Privacy Policy

What This Tool Is

MooseGlove is an open-source portfolio project. It is not a commercial service and does not collect personal information for marketing, advertising, or resale.

Data You Submit

When you submit a clinical scenario through the interface, the following data is sent to the backend API:

  • Symptoms, age, sex, medical history, medications, vital signs, and lab values you enter
  • Free-text clinical notes (if provided)

This data is processed in real time to generate a differential diagnosis. It is not stored in a database, not written to disk, and not retained after the request completes. The backend does not have a user account system and does not associate requests with identifiable individuals.

Do Not Submit Real Patient Data

This is a demonstration tool. Do not enter identifiable patient information (names, dates of birth, medical record numbers, addresses, or any other protected health information). The system is not designed to comply with HIPAA, GDPR health data requirements, or equivalent regulations.

Observability and Logging

The backend logs structured events (request latency, error codes, model used) via Langfuse for performance monitoring. These logs contain request metadata, not the clinical content you submit. No symptoms, diagnoses, or patient data are written to logs.

Analytics

The frontend uses Cloudflare Web Analytics, a privacy-focused analytics service that does not use cookies, does not track users across sites, and does not collect personally identifiable information. It records aggregate page views and performance metrics only.

Cookies and Local Storage

MooseGlove does not set cookies. It uses browser local storage to remember your theme preference (light or dark mode) and sidebar layout state. No authentication tokens or tracking identifiers are stored.

Third-Party Services

  • Cloudflare (CDN, DNS, tunnel, Web Analytics)
  • Groq / Cerebras (LLM inference, receives the clinical scenario for processing)
  • Langfuse (backend performance monitoring, no clinical data)

Each service has its own privacy policy. The LLM providers (Groq, Cerebras) receive the clinical scenario text as part of the inference request. Consult their privacy policies for details on how they handle input data.
